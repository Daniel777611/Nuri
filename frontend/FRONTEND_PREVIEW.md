# 前端页面预览与设计还原

## 启动

在 `frontend` 目录运行：

```bash
npm run preview
```

然后打开 <http://localhost:8081>。代码保存后页面会自动刷新；按 `Ctrl+C` 停止服务。若样式或路由缓存异常，使用 `npm run preview:clear`。

项目已在本地配置 Node 20 运行时（`.tools/node20`），因此不依赖电脑全局 Node 版本。该命令默认开启**演示模式**：所有界面会使用本地样例数据，不请求后端；注册页填写任意合法邮箱及至少 6 位密码即可继续浏览完整流程。

## 检查页面与点击交互

- 页面文件位于 `app/`；文件路径就是路由。例：`app/(tabs)/tasks.tsx` 对应任务页，`app/detail/[id].tsx` 对应详情页。
- 在浏览器中点击、填写表单、切换 tab，即可验证实际交互和跳转；浏览器控制台会显示前端报错。
- 打开浏览器开发者工具的设备模拟（iPhone 14：390 × 844），即可按移动端尺寸检查。建议把缩放设为 100%。

## 与设计稿逐项比对

1. 将预览窗口固定为设计稿的画板尺寸（通常为 390 × 844）。
2. 截取页面，或把设计稿作为透明图层叠在浏览器截图上，比对边距、字号、圆角、颜色和阴影。
3. 全局视觉 token 优先在 `src/theme.ts` 调整；页面级布局调整对应 `app/` 下的页面文件；可复用卡片与弹窗在 `src/components/`。
4. 每次改动后同时检查默认、按下/点击、空状态和加载/报错状态（如存在），避免只对齐静态画面。
