## 3. 产品最新定义

### 3.1 最新一句话定义

```text
Family Growth Radar 是一个面向父母的儿童成长阶段行动雷达。
它基于儿童发展循证知识库和长期家庭画像，
根据孩子年龄、发展状态、家庭背景和父母回应模式，
主动提醒父母每个阶段该做什么、为什么现在重要、怎么做、哪些不能错过、什么时候需要专业人士。
```

### 3.2 产品不是

Family Growth Radar 不是：

- 普通育儿 App
- AI 育儿聊天机器人
- ChatGPT for parenting
- 儿童内容平台
- AI 陪伴玩具
- 把网上育儿信息重新包装
- 医疗诊断工具
- 心理诊断工具
- 临床治疗工具
- 制造父母焦虑的“错过窗口”恐吓产品

### 3.3 产品真正提供的价值

父母表面上会说：

- 我想知道宝宝 6 个月要做什么
- 网上说法太多，我不知道信谁
- 我知道要做，但不知道怎么做
- 我不知道哪些是必须做，哪些只是建议
- 我家孩子情况和别人不一样

Family Growth Radar 真正提供的是：

```text
主动、循证、阶段化、可执行、可反馈的家庭成长行动指导。
```

它不是等待父母提问，而是主动判断孩子当前成长窗口，提醒父母当下最值得知道、准备、观察和执行的事情。

### 3.4 内部战略口号

```text
在正确的时间，做正确的事，并用正确的方式做。
```

含义：

- 正确的时间：不让父母错过关键成长窗口
- 正确的事：基于权威指南、儿童发展理论和专业共识
- 正确的方式：用高关爱 + 高结构的权威型方式完成成长任务

## 4. 核心洞察

### 4.1 父母不知道自己不知道什么

当前最重要的产品洞察是：

```text
父母最大的问题不是没有爱，也不是完全没有信息，
而是不知道孩子每个阶段有哪些自己应该提前知道、准备、观察和执行的关键行动。
```

这可以称为：

```text
Anticipatory Guidance Gap
```

也就是父母缺少及时、可信、可执行的阶段性预期指导。

### 4.2 为什么普通 ChatGPT / 搜索不够

普通 ChatGPT 或搜索的问题：

- 父母想起来才问
- 输出通常是通用答案
- 信息来源混杂
- 不追踪父母是否执行
- 不知道家庭上下文
- 不区分 Must Know / Must Do / Watch / Ask Professional
- 不长期记录父母回应模式

Family Growth Radar 的差异：

- 系统按孩子年龄和状态主动提醒
- 先经过专业知识库过滤
- 根据家庭画像和孩子阶段个性化排序
- 连续追踪执行反馈
- 区分重要级别
- 帮助家庭走向高关爱 + 高结构

### 4.3 长期护城河

长期壁垒不是育儿内容本身，而是：

```text
儿童发展循证知识库
+ 长期孩子画像
+ 家庭情境理解
+ 父母回应画像
+ 行动反馈数据
+ 信任关系
+ 专业边界
+ 跨文化家庭语境
```

如果系统真正长期理解一个孩子和一个家庭，switching cost 会很高。

## 5. 产品底层模型

当前产品底层模型是三层：

```text
Evidence Knowledge Base
+ Stage Radar
+ Action Translator
```

### 5.1 Evidence Knowledge Base

这是产品的知识边界和可信度基础。

它不是普通内容库，不是文章库，也不是让 AI 随意抓取互联网。

它应该存储：

- 权威机构指南
- 儿童发展心理学理论
- 专业共识
- 研究证据
- 内部转译规则
- 红旗边界
- 输出语气规则

核心原则：

```text
AI 是转译器和个性化调度器，不是知识源头。
知识源头必须是团队自己的儿童发展循证知识库。
```

### 5.2 Stage Radar

Stage Radar 的职责：

- 判断孩子当前处于哪个成长窗口
- 识别父母本阶段最容易错过的关键行动
- 区分必须知道、必须做、需要观察、可选增强、需要问专业人士
- 把信息压缩成父母本周可以完成的 1-3 个行动
- 根据家庭背景和父母反馈持续调整提醒

### 5.3 Action Translator

Action Translator 把专业知识转译成父母能执行的内容：

- Growth Window Map
- Weekly Action Brief
- Action Card
- 父母话术
- 观察指标
- 医生问题清单
- 下周反馈问题

输出必须做到：

- 温和
- 具体
- 不恐吓
- 不羞辱
- 不诊断
- 不替代专业人士
- 每次只给 1-3 个关键行动

## 6. 儿童发展循证知识库设计

### 6.1 四层知识结构

Level 1：权威机构指南层

- CDC
- AAP / Bright Futures
- WHO
- 儿科健康监督
- 营养和发展里程碑官方指南

用途：

- Must Know
- Must Check
- Ask Professional

Level 2：儿童发展心理学理论层

- 依恋理论
- 气质理论
- 社会文化发展
- 生态系统理论
- 情绪调节
- 家庭系统
- 权威型教养

用途：

- 解释框架
- 家庭互动判断
- 父母回应方式建议

Level 3：研究证据与专业共识层

- 同行评议研究
- 专业组织共识
- 研究综述
- responsive feeding
- 亲子阅读
- 双语语言发展

用途：

- 补充权威指南不够细的场景
- 但必须标注证据强弱

Level 4：产品转译层

- 成长窗口
- 行动卡
- 观察清单
- 父母话术
- 儿科 visit 问题清单
- 红旗转介边界

用途：

- 父母真正看到和执行的内容

### 6.2 Knowledge Item 字段

建议字段：

- Knowledge Item ID
- Topic
- Age Range
- Development Domain
- Source Type
- Source Name & URL
- Evidence Level
- Consensus Level
- Parent Action Type
- Urgency
- Personalization Variables
- Contraindications / Exceptions
- When to Refer
- Parent-facing Explanation
- Internal Expert Notes
- Last Reviewed / Reviewer

### 6.3 证据等级

A：权威机构明确建议

- 可以做强提醒
- 仍需说明个体差异和例外

B：专业共识强，具体做法需个性化

- 表达为“通常建议”“多数家庭可以尝试”

C：研究支持，但不应过度外推

- 表达为“有研究支持这个方向”“可以低风险尝试”

D：发展心理学解释框架

- 用于解释和观察
- 不能直接当硬性指令

E：基于用户个案信息的系统推断

- 必须加限制
- 明确非诊断

## 7. 当前 MVP：6-Month Growth Window Pilot

### 7.1 MVP 名称

```text
6-Month Growth Window Pilot / 6 个月成长窗口提醒试点
```

### 7.2 MVP 用户

当前建议从更小切口开始：

```text
孩子 5-7 个月的美国华人家庭
```

这是从原先“0-3 岁家庭”进一步收窄后的第一阶段 wedge。

原因：

- 6 个月左右存在清晰成长窗口
- 辅食启动是父母普遍关心且容易信息混乱的任务
- 父母通常愿意分享具体情况
- 内容可以连接儿科建议、发展观察、家庭 routine、父母回应方式
- 便于做 4 周 pilot 和付费测试

### 7.3 MVP 核心承诺

```text
帮父母在宝宝进入 6 个月阶段时，
知道现在该做什么、为什么重要、怎么开始、哪些不用焦虑、哪些需要问医生。
```

### 7.4 MVP 周期

建议周期：

```text
4 周
```

### 7.5 MVP 样本量

建议第一轮：

```text
3-5 个家庭
```

### 7.6 价格测试

建议可测试：

```text
$49 / $99 / $199
```

前 3 个家庭可以 founder discount，但建议不要完全免费。

原因：

- 如果完全免费，很难判断付费意愿
- 付费哪怕很小，也比口头“有用”更能说明价值
- 可以让父母更认真完成反馈

### 7.7 成功标准

第一轮通过标准：

```text
3-5 个家庭完成 pilot
至少 2 个家庭连续反馈
至少 2 个家庭愿意付费继续
至少 1 个家庭愿意推荐
没有明显焦虑型负反馈
```

### 7.8 四周交付流程

Week 0：Intake

- 30 分钟 intake
- 收集喂养、睡眠、语言、家庭背景、儿科 visit、父母担心点
- 验证父母是否愿意分享家庭上下文

Week 1：6-Month Growth Map + 辅食准备行动卡

- 说明为什么 6 个月窗口重要
- 给出本周 1-3 个行动
- 验证提醒是否及时、有用、可信

Week 2：喂养反馈 + 互动语言输入 + 大运动观察卡

- 追踪是否执行
- 记录孩子反应
- 记录父母遇到的摩擦点

Week 3：睡眠 / 安全 / routine + 权威型照护方式微调

- 把行动指导连接到家庭节奏
- 加入高关爱 + 高结构的父母回应方式

Week 4：阶段复盘报告 + 7-9 个月窗口预告

- 总结已完成任务
- 指出下一个成长窗口
- 测试是否愿意继续付费
- 测试是否愿意推荐朋友

## 8. 6 个月辅食窗口示例

### 8.1 本周成长重点

辅食准备不是为了马上吃很多，而是让孩子开始练习：

- 探索食物
- 吞咽
- 口腔运动
- 家庭餐参与
- 饥饿和饱足信号表达
- 父母 responsive feeding

### 8.2 本周结构

父母可以做：

- 准备安全餐椅
- 每天选择稳定时间尝试
- 从少量、合适质地开始
- 保持观察记录
- 整理儿科医生问题

### 8.3 关爱回应

建议语气：

```text
观察宝宝饥饿和饱足信号。
不强迫、不恐吓、不用屏幕哄喂。
宝宝拒绝时保持平静，明天再试。
```

父母话术示例：

```text
你可以尝一尝。
不想吃也没关系，我们明天再试。
```

### 8.4 观察指标

可以观察：

- 坐姿支持
- 头颈控制
- 对食物兴趣
- 吞咽
- 咳呛
- 皮疹
- 便秘
- 进食情绪

### 8.5 何时问医生

如果出现以下情况，建议先与儿科医生确认：

- 早产
- 体重增长问题
- 严重湿疹 / 过敏史
- 吞咽困难
- 频繁呛咳
- 明显发育担忧

必须避免的说法：

```text
晚加辅食一定导致语言迟缓。
```

更稳妥的说法：

```text
延迟辅食或质地进阶可能影响营养、进食技能和口腔运动经验；
语言发展受多因素影响，不能单一归因。
```

## 9. 权威型家庭目标模型

### 9.1 产品目标

Family Growth Radar 的家庭教育目标不是让父母更会控制孩子，而是帮助家庭形成：

```text
温暖而有边界的回应系统
```

也就是：

```text
高关爱 + 高结构
```

### 9.2 四种家庭回应模式

权威型：

- 高关爱 + 高结构
- 父母温暖、回应、支持
- 同时设定清晰稳定的限制
- 这是产品希望帮助家庭靠近的方向

专断型：

- 低回应 + 高控制
- 规则强，但孩子感受常被忽略
- 产品应增加情绪确认、解释、选择和事后修复

放纵型：

- 高关爱 + 低结构
- 接纳多，但边界不稳定
- 产品应增加少量清晰规则、有限选择和稳定执行

拒绝 / 忽视型：

- 低关爱 + 低结构
- 既缺少回应，也缺少稳定边界
- 产品应先建立基本回应、连接、routine 和安全支持

### 9.3 建议输出标准格式

每条建议都应同时包含：

```text
情绪回应 + 清晰边界
```

例如：

1. 看见情绪：我知道你还想继续玩。
2. 说明边界：但是现在到回家的时间了。
3. 给有限选择：你可以自己走到门口，或者我抱你走。
4. 稳定执行：你很难过，我会陪你，但我们还是要回家。
5. 事后连接：刚才很难，但我们一起完成了。

### 9.4 产品语气规则

不要说：

```text
你是放纵型父母。
```

更好的说法：

```text
在睡前场景里，你们家目前更容易出现“关爱很多，但边界不稳定”的模式。
本周我们只调整一个点：先确认情绪，再坚持边界。
```

## 10. 0-2 岁依恋关系知识模块

### 10.1 为什么加入依恋模块

Family Growth Radar 面向 0-3 岁家庭，而 0-2 岁是孩子与主要照护者形成依恋关系的关键时期。

依恋关系不是让孩子“更黏父母”，而是让孩子形成一个稳定预期：

```text
当我需要你时，你大体上会回应我；
当我害怕、饿、累、痛苦时，我可以向你靠近；
这个世界虽然有不舒服，但我不是孤立无援的。
```

### 10.2 0-2 岁依恋发展阶段

0-2 个月：前依恋阶段

- 对人声、气味、触摸有反应
- 尚未稳定区分主要照护者

2-6 个月：依恋形成中

- 开始偏好熟悉照护者
- 用微笑、眼神、哭声、身体放松建立互动

6-18 个月：清晰依恋阶段

- 分离焦虑、陌生人焦虑出现
- 父母成为安全基地
- 孩子一边探索世界，一边确认父母还在

18-24 个月：目标协调阶段

- 逐渐理解父母会离开也会回来
- 开始用语言、协商、等待维持安全感

### 10.3 四种依恋类型

安全型依恋：

- 父母大体稳定、敏感、可修复
- 孩子受挫时会求助
- 被安抚后可以重新探索

回避型依恋：

- 父母常冷淡、拒绝或嫌烦
- 孩子学会少表达需求
- 表面上看似独立，但可能是需求被压住

焦虑 / 矛盾型依恋：

- 父母回应不稳定、不可预测
- 孩子黏人、警觉、难以重新探索
- 想靠近又难以被安抚

紊乱型依恋：

- 照护关系让孩子同时想靠近又害怕
- 可能出现僵住、退缩、混乱反应
- 通常需要专业支持

### 10.4 产品化含义

Family Growth Radar 不应该给父母或孩子贴“依恋类型”标签。

更适合的产品目标是识别具体场景中的照护循环：

```text
孩子发出信号 -> 父母看见 -> 父母回应 -> 孩子被安抚 -> 孩子重新探索
```

产品可以帮助父母：

- 看见孩子信号
- 区分需求和行为表现
- 更稳定回应
- 减少忽略、羞辱、威胁、过度控制
- 在情绪上接住孩子
- 在行为上建立结构

## 13. 安全、隐私、合规边界

### 13.1 医疗与发展边界

产品可以做：

- 正常发展范围内的阶段提醒
- 父母行动建议
- 家庭互动建议
- 黄灯情况的观察记录
- 建议在儿科 visit 中讨论
- 红灯情况建议联系专业人士

产品不能做：

- 承诺保证发展结果
- 判断孩子是否自闭症 / ADHD
- 诊断发育迟缓
- 提供临床治疗方案
- 替代儿科医生
- 替代儿童心理或发展专业人士
- 提供紧急处理指令

### 13.2 Green / Yellow / Red Zone

Green Zone：

- 正常发展范围内的阶段提醒、父母行动、家庭互动建议
- 不承诺结果

Yellow Zone：

- 提示持续观察
- 建议记录并在儿科 visit 中讨论
- 不包装成诊断

Red Zone：

- 明确建议联系儿科医生、发展专家、儿童心理/临床专业人士或 emergency support
- 不提供替代医疗、心理治疗或紧急处理指令

### 13.3 儿童数据与隐私原则

必须坚持：

- 最小必要收集
- 清晰同意
- 不默认用于模型训练
- 删除权
- 访问控制
- 敏感信息谨慎
- 儿童照片、视频、医疗信息、诊断信息尽量避免或单独强化保护
- 不把“权威来源”说成“产品已获权威机构认证”

### 13.4 不可说清单

不能说：

- 我们诊断孩子发育问题
- 我们判断你的孩子是否自闭症 / ADHD
- 错过辅食一定导致语言迟缓
- 照我们做就能保证孩子发展正常
- 我们的建议替代儿科医生
- 权威机构认证了我们的产品，除非事实获得认证
