## 3. 项目一句话定义

当前更准确的一句话定义：

```text
Family Growth Radar 是一个面向家庭的个性化成长顾问系统，长期目标是让每个家庭都拥有一位既专业、又真正了解自己孩子的成长顾问，帮助父母在日常生活中做出更精准、更适合这个孩子的教育决策。
```

早期阶段的一句话定义：

```text
先帮助 0-3 岁孩子父母知道孩子当前成长阶段应该做什么、观察什么、如何陪伴，并验证这些阶段行动提示是否真的有帮助、是否愿意持续接收、是否愿意付费。
```

## 4. 终局愿景

### 4.1 终局不是普通育儿 App

这个产品最终不是：

- 普通宝宝记录 App
- AI 育儿问答工具
- 育儿知识库
- AI 陪孩子聊天的产品
- AI 教孩子学习的产品
- 儿童测评打分工具
- 医疗诊断工具
- 心理治疗工具
- 监控孩子的智能家居系统

### 4.2 终局是什么

终局愿景：

```text
成为每个家庭中一位专业、长期在场、真正了解这个孩子的个性化家庭成长顾问。
```

它应当能够：

- 了解这个孩子当前处于什么发展阶段
- 了解这个孩子的气质、节律、兴趣、情绪和行为模式
- 了解这个家庭的照护方式、语言环境、文化背景和教育冲突
- 解释孩子行为背后的发展意义
- 提醒父母这个阶段该观察什么、支持什么、避免误读什么
- 给出针对这个孩子、这个家庭、这个阶段的行动建议
- 跟踪建议是否被执行
- 根据反馈不断调整孩子画像和指导方案
- 必要时提醒父母寻求儿医、儿童发展专家、心理健康专业人士或其他专业支持

### 4.3 最核心的差异化

普通育儿产品回答：

```text
3 岁孩子发脾气怎么办？
```

本产品最终要回答：

```text
你家这个孩子最近为什么更容易在睡前崩溃？
结合过去两周的睡眠、白天活动、父母回应、屏幕时间和分离焦虑信号，这周你们最应该先调整什么？
```

区别在于：

- 不是泛泛回答一个年龄段
- 而是长期理解一个具体孩子
- 不是提供知识
- 而是提供针对这个家庭的可执行方案

## 5. 产品底层逻辑

### 5.1 长期闭环

长期产品闭环：

```text
专业发展模型
  -> 引导父母观察关键发展信号
  -> 结合家庭日常数据和未来智能家居信号
  -> 建立孩子动态成长画像
  -> 生成个性化教育/陪伴/环境调整方案
  -> 父母执行
  -> 系统复盘调参
  -> 更深入地认识这个孩子
```

### 5.2 护城河

本项目真正的护城河不是“育儿内容多”。

真正护城河是：

```text
专业观察能力 + 长期个体画像 + 家庭场景闭环
```

具体来说：

- 系统知道不同年龄阶段应该观察什么
- 系统能把专业观察问题翻译成父母能回答的日常问题
- 系统能持续沉淀这个孩子的行为模式和家庭背景
- 系统能把这些信息转化成更精准的指导
- 系统能通过复盘让建议越来越贴近这个孩子

### 5.3 为什么需要智能家居

未来接入智能家居的意义不是“监控孩子”。

意义是：

```text
让系统低负担地看见家庭日常模式，从而更好理解孩子的真实生活节律。
```

可能的数据或信号：

- 睡眠与作息
- 屏幕使用时段
- 环境噪音
- 灯光节律
- 家庭日历
- 照护者变化
- 活动水平
- 家庭提醒与仪式

智能家居必须遵守边界：

- 不把孩子变成被监控对象
- 不做诊断
- 不做打分
- 不公开家庭隐私
- 父母必须明确授权
- 能删除、能导出、能关闭

## 6. 当前阶段判断

### 6.1 现在不做什么

当前不应该马上做：

- 完整 iOS App
- 完整 Web App
- 大规模机器人群服务
- 自动化推送系统
- 复杂孩子画像
- 智能家居接入
- 医疗报告上传
- OCR
- 疫苗规则库全球化
- 复杂账户系统
- 付费系统

原因：

```text
核心价值还没有被真实家庭验证。
过早开发会把未经验证的流程固化。
```

### 6.2 当前最应该做什么

当前最应该做的是：

```text
阶段 1：贵宾式 MVP
```

用人工方式服务 1-3 个美国华人 0-3 岁孩子家庭，验证：

1. 父母是否认为当前阶段行动提示有实际帮助
2. 父母是否愿意持续接收
3. 父母是否愿意为更稳定或更个性化的服务付费

## 7. 两阶段 MVP 策略

当前明确采用：

```text
先深后广
```

### 7.1 阶段 1：贵宾式 MVP

目标：

```text
服务 1-3 个家庭，用人工方式验证行动指导是否真的有帮助。
```

形式：

- 人工挑选家庭
- 人工收集基础信息
- 人工生成行动指南
- 人工发送
- 人工复盘

验证问题：

- 这份指导是否真的有帮助
- 家长是否愿意继续接收
- 家长是否愿意付费或了解价格
- 内容是否太泛泛、太焦虑、不适合

进入下一阶段门槛：

```text
至少 1-2 个家庭明显觉得有帮助
至少 1 个家庭执行了一个建议动作
至少 1 个家庭愿意继续
至少 1 个家庭表达付费意愿或愿意了解价格
没有明显焦虑型负反馈
```

### 7.2 阶段 2：自动化群服务 MVP

目标：

```text
服务约 100 个家庭，验证年龄段分层的阶段行动指导是否能规模化触达、阅读、互动和持续使用。
```

形式：

- 自动化机器人
- 群或频道承载
- 按年龄段分层
- 轻量反馈
- 不承诺深度个性化

验证问题：

- 群场景是否成立
- 年龄段分层内容是否够用
- 自动化推送是否被阅读
- 是否有人互动
- 哪些家庭愿意进入深度个性化服务

注意：

```text
阶段 2 不应该假装“已经很懂每个孩子”。
它只是规模化触达和筛选深度用户的入口。
```

## 8. 阶段 1：贵宾式 MVP 详细执行方案

### 8.1 阶段 1 一句话目标

```text
验证 0-3 岁孩子父母收到“当前阶段行动指导”后，是否认为这些提示有实际帮助，是否愿意持续接收，并是否愿意为更稳定或更个性化的指导付费。
```

### 8.2 目标家庭

第一阶段目标用户：

```text
美国华人家庭，孩子年龄 0-3 岁。
```

优先选择：

- 你认识或半熟关系的家庭
- 愿意真实反馈
- 孩子 0-3 岁
- 父母愿意提供极简信息
- 父母愿意 7 天后复盘
- 最好不是极端复杂或高风险情况

暂时避开：

- 严重医疗问题
- 已明确发育迟缓诊断但需要专业干预的情况
- 父母正在经历严重心理危机
- 高冲突家庭法律/监护问题
- 需要即时医疗或心理安全介入的案例

原因：

```text
第一轮验证的是行动指导价值，不是复杂危机干预能力。
```

### 8.3 第一轮样本量

建议：

```text
1-3 个家庭
```

不要一开始做 10-15 个家庭。

原因：

- 当前内容模板还没打磨
- 样本太多会增加人工压力
- 第一轮目标不是统计显著
- 第一轮目标是看服务形态有没有生命力

### 8.4 邀请话术

建议邀请话术：

```text
我现在在测试一个给 0-3 岁孩子父母用的“当前阶段行动指南”。

你只需要告诉我孩子年龄和一点基本情况，我会人工给你整理一份：
这个阶段孩子可能在发展什么、这周父母可以做什么、建议观察什么、一个很简单的陪伴动作。

这不是医疗诊断，也不会评价孩子，只是帮父母更清楚当前阶段可以怎么陪伴和回应。

7 天后我会问你几个反馈问题，看它有没有帮助。
```

### 8.5 边界说明

必须在用户参与前说明：

```text
这个实验不会做医疗诊断、心理诊断，也不会判断孩子正常或异常。
如果涉及疾病、发育迟缓、过敏、用药、急症等问题，请以儿医或专业人士意见为准。
你提供的信息只用于这次行动指南实验，不会公开分享。你可以随时要求删除。
```

### 8.6 极简信息收集

不要做长问卷。

第一轮只收集：

```text
孩子出生日期 / 当前年龄：
孩子性别，可选：
所在城市/州，可选：
孩子昵称，可选：
有没有你觉得需要补充的情况？可自由写：
最近你最不确定“该怎么做”的地方是什么？可留空：
你希望通过什么方式收到？微信 / 短信 / 邮件 / WhatsApp / 其他：
```

这里的关键是：

```text
自由输入优先于复杂结构化表单。
```

原因：

- 父母未必知道哪些信息重要
- 过长表单会像测评或病历
- 第一阶段不需要完整孩子画像
- 自由输入更容易暴露真实困惑

### 8.7 行动指南模板

每个家庭收到一份不超过一页的行动指南。

模板：

```text
孩子当前阶段：

1. 这个阶段孩子可能正在发展什么

2. 这周父母最可以做的 3 件事

3. 本周建议观察的 3 个信号

4. 今天可以做的 1 个小动作

5. 父母容易误读的地方

6. 可以准备的环境或材料

7. 什么时候需要咨询专业人士
```

### 8.8 内容语气原则

推荐使用：

```text
这个阶段可以开始观察……
这周可以试着……
如果你持续担心，可以记录下来，和儿医或专业人士讨论。
今天只做一个很小的动作就好。
```

避免使用：

```text
你的孩子应该会……
必须……
错过关键期……
落后……
训练孩子……
不做会影响一生……
```

内容应该：

- 温和
- 具体
- 可执行
- 不制造焦虑
- 不评价孩子
- 不责备父母
- 不把孩子行为解释成“不听话”“不配合”
- 涉及健康时提示咨询专业人士

### 8.9 7 天后复盘问题

7 天后问：

```text
1. 这份指南有没有让你觉得有帮助？0-10 分
2. 哪一部分最有帮助？
3. 你有没有做里面哪个动作？做了哪个？
4. 哪一部分太泛泛、不适合，或让你有压力？
5. 你愿意下周继续收到类似指导吗？
6. 如果之后每周都能收到类似的阶段行动指导，并且逐渐更贴合你家孩子，你愿意为这个服务付费吗？
7. 如果愿意，你觉得什么价格是可以接受的？
```

注意：

```text
口头付费意愿是弱信号。
愿意真实付费继续第二周是强信号。
```

### 8.11 第一阶段通过标准

1-3 个家庭里：


### 8.12 第一阶段产出

必须产出：

- 真实家庭反馈
- 家长原话
- 哪些建议容易被执行

## 9. 阶段行动指南内容设计

### 9.1 年龄段划分

当前建议粗分：

```text
0-3 个月
4-6 个月
7-12 个月
13-18 个月
19-24 个月
25-36 个月
```

这是为了：

- 降低内容复杂度
- 避免一开始就做精细发展里程碑库
- 足够支撑第一轮人工服务
- 后续可以根据反馈拆细

### 9.2 0-3 个月重点

常见主题：

- 睡眠节律初建
- 喂养节奏
- 哭声和需求识别
- 亲密照护
- 安全感
- 父母恢复和压力

行动指导方向：

- 建立温和可重复的安抚流程
- 观察孩子什么时候更容易平静
- 父母不要把哭理解成“被宠坏”
- 注意安全睡眠
- 涉及喂养、体重、发热等咨询儿医

### 9.3 4-6 个月重点

常见主题：

- 互动增加
- 翻身和身体探索
- 手眼协调
- 睡眠变化
- 可能开始准备辅食

行动指导方向：

- 给孩子安全的地面活动时间
- 观察孩子如何主动看、抓、发声
- 不急着训练动作达标
- 辅食准备以儿医建议为准

### 9.4 7-12 个月重点

常见主题：

- 爬行或移动探索
- 分离焦虑
- 辅食体验
- 抓、扔、咬、拍
- 模仿和社交回应

行动指导方向：

- 提供安全探索环境
- 观察孩子如何表达拒绝、想要、求助
- 不把扔东西等探索行为简单理解为捣乱
- 辅食反应只记录，不诊断

### 9.5 13-18 个月重点

常见主题：

- 自主性增强
- 语言理解发展
- 情绪表达强烈
- 走、拿、探索、模仿
- 规则边界开始重要

行动指导方向：

- 给有限选择
- 用短句和稳定回应
- 观察触发情绪爆发的场景
- 不把自主探索直接理解为“不听话”

### 9.6 19-24 个月重点

常见主题：

- 语言爆发或表达受限带来的挫败
- 情绪调节能力不足
- 秩序感和坚持
- 分享困难
- 屏幕和边界冲突

行动指导方向：

- 帮孩子命名情绪
- 提前预告转换
- 保持规则一致
- 用动作替代长篇说教
- 观察哪些边界最容易冲突

### 9.7 25-36 个月重点

常见主题：

- 自我意识
- 假装游戏
- 社交冲突
- 如厕准备
- Daycare 适应
- 屏幕、规则、拖延

行动指导方向：

- 用游戏和仪式帮助过渡
- 关注孩子是否能表达需求
- 帮父母设计一致的回应脚本
- 不把情绪爆发简单理解为“故意”
- 如厕不要过度强迫

## 13. 隐私、安全、合规边界

### 13.1 产品必须避免

不得：

- 做医疗诊断
- 做心理诊断
- 判断孩子正常/异常
- 判断发育迟缓结论
- 判断药物剂量
- 判断疫苗禁忌
- 判断过敏严重程度
- 给孩子打分
- 制造“错过关键期”焦虑
- 暗示父母不做某事会毁掉孩子未来
- 把孩子行为简单归因于“不听话”“不配合”“坏习惯”

### 13.2 高风险情况处理

如果用户描述：

- 呼吸困难
- 严重过敏迹象
- 持续高热
- 抽搐
- 明显脱水
- 意识异常
- 严重外伤
- 父母有自伤/伤害孩子风险

应回应：

```text
这个描述可能涉及紧急健康/安全风险。请立即联系当地急救服务、儿医或就近医疗机构。
我不能在这里判断原因或给出治疗建议。
我可以帮你整理已经观察到的信息，方便你向医生说明。
```

### 13.3 第一阶段隐私说明

对参与家庭说明：

```text
你提供的信息只用于本次行动指南实验。
不会公开分享。
不会用于训练模型。
你可以随时要求删除。
我们不会做医疗或心理诊断。
```

### 13.4 美国市场注意

后续如果进入产品化，需要评估：

- COPPA
- HIPAA 适用性
- FTC 健康 App 指南
- 州隐私法
- 儿童数据保护
- 明确监护人同意
- 数据删除与导出

阶段 1 仍应坚持数据最小化。

## 14. 付费假设

### 14.1 当前要验证的付费问题

不是验证完整商业模式。

只验证：

```text
父母是否愿意为持续、稳定、更贴合自己孩子的阶段行动指导付费。
```

### 14.2 付费信号强弱

弱信号：

- 说“听起来不错”
- 说“以后可能会付费”
- 愿意了解价格

中信号：

- 明确说愿意付费
- 给出可接受价格
- 愿意继续第二周

强信号：

- 真实付费继续第二周
- 主动要求继续
- 推荐给其他家庭

### 14.3 阶段 1 可问的付费问题

```text
如果之后每周都能收到类似的阶段行动指导，并且逐渐更贴合你家孩子，你愿意为这个服务付费吗？

如果愿意，你觉得什么价格是可以接受的？
```

更强测试：

```text
我们下一周可以继续为你做一份更针对你家孩子的行动指南，早期测试价格是 $X / 周，你愿意继续吗？
```

### 14.4 不要过早定价

阶段 1 可以探索价格区间，但不要过早确定订阅价。

可能探索：

- 单次行动指南
- 2 周陪伴包
- 月度顾问包
- 专家复核版本
